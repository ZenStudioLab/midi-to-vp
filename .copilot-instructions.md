# MIDI to VP - Copilot Web Code Review Instructions

Use this file for GitHub Web Copilot PR review in `midi-to-vp/`.

## Context Order

1. `AGENTS.md`
2. `README.md`
3. `docs/architecture.md`
4. `docs/adr/README.md` and relevant ADRs
5. Changed source and tests

## Priority Checks

1. Parsing/conversion correctness and edge cases
2. Deterministic output behavior across notation modes
3. Error handling for malformed inputs
4. Type safety and API contract stability
5. Test updates for newly introduced paths and edge cases

## Output Format

Report findings by severity: `P0`, `P1`, `P2`.
Each item includes file+line, risk, and minimal fix.
If clean, state: `No blocking issues found` with testing gaps.

## Review Style

- Focus on functional correctness and reliability.
- Avoid non-functional style commentary.
- Prefer minimal targeted fixes.
